# A4: ручной процент сделки и адаптивная сетка

Дата утверждения: 2026-07-02.

## Цель

Заменить неоднозначный «Расчётный задаток» прямым ручным процентом для конкретной сделки и устранить обрезание/разнобой полей на мобильном и десктопном экранах.

## Расчётный контракт

- Новое поле строки: `dealManualRates`.
- Пустое значение означает автоматическую шкалу.
- `50` означает ровно `50%`, `65` — `65%`, `80` — `80%`.
- Ручной процент действует только на текущую строку.
- Ручной процент не изменяет квалификацию строки и автоматический счётчик следующих сделок.
- Обычная сделка меньше `50 000 ₽` всегда получает `45%`, не квалифицируется и игнорирует ручной процент.
- Новостройка с одним агентом квалифицируется при любой сумме и допускает ручной процент.
- В стандартной и повышенной системе ручной процент разрешён.
- В фиксированной системе поле не показывается, действует общий `fixedRate`.
- Допустимый диапазон ручного процента: `0–100`; пусто сохраняется как пустая строка.

## Миграция

Старый `dealDepositOrders` читается только как legacy-источник, когда у строки отсутствует новое значение:

| Старый номер | Новый процент |
|---:|---:|
| 1 | 45 |
| 2 | 50 |
| 3 | 55 |
| 4 | 60 |
| 5 | 65 |
| 6 | 70 |
| 7 и больше | 80 |

После нормализации расчёт, UI и новые сохранения используют `dealManualRates`. Старые черновики и snapshot V1/V2/V3 не теряют эквивалентный результат.

## UI

- Подпись: «Процент для этой сделки, %».
- Подсказка: «Пусто — автоматическая шкала. Меняет только эту сделку».
- Для обычной строки меньше `50 000 ₽` поле disabled и показывается пояснение про обязательные `45%`.
- Для fixed поле не рендерится.
- Источник ставки отображается как «Ручной процент».
- На десктопе сумма, ручной процент и условие новостройки образуют одну сетку с одинаковой структурой: заголовок, контрол, подсказка.
- На мобильном сетка становится одной колонкой `minmax(0, 1fr)`.
- У grid-элементов и контролов задаются `min-width: 0`/`max-width: 100%`, чтобы `overflow: hidden` карточки не обрезал поля.

## Интеграции

- A4 draft V2 сохраняет новый массив внутри месячного state.
- `domianA4TableSnapshot` передаёт только активный месяц и новый массив.
- `table-ledger.js` использует `manualRate` в модели сделки, умеет читать legacy `dealDepositOrders` и сохраняет новое значение в расчётный агент.
- `table-mode.js` сохраняет паритет расчёта и читает V1/V2/V3.

## Проверка

- `manualRate: 50` даёт ставку `0.50`, а не `0.80`.
- Ручная ставка не двигает следующую автоматическую ступень.
- Маленькая обычная сделка игнорирует ручную ставку.
- Маленькая новостройка применяет ручную ставку.
- Fixed игнорирует ручную ставку.
- Legacy-номера мигрируют по утверждённой таблице.
- Массивы остаются выровненными при добавлении/удалении строк.
- Статические UI/CSS-тесты закрепляют подписи и responsive-ограничения.
